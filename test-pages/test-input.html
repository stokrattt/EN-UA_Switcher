Switcher — Browser Input Test Page

Use this page to test Switcher's UI Automation adapter with browser inputs. Launch Switcher.App first, then test each field below.

Hotkeys: Ctrl+Shift+K = fix last word  |  Ctrl+Shift+L = fix selected text

1. Input (type="text")

Expected adapter: UIAutomationTargetAdapter (ValuePattern). Should support safe last word and auto mode.

2. Textarea

Expected adapter: UIAutomationTargetAdapter (ValuePattern). Known limitation: cursor may jump to end after replacement.

3. ContentEditable div

⚠ Expected result: UNSUPPORTED. Switcher should detect no writable ValuePattern, log "unsupported target", and do nothing. Text must not be changed or erased.

Test procedure

  1. Launch Switcher.App.exe
  2. Open this page in Chrome or Edge
  3. Focus the input[type=text] field, type ghbdsn
  4. Press Ctrl+Shift+K
  5. Verify the text changes to привіт
  6. Repeat for each test case in the table
  7. Focus the contenteditable div and press Ctrl+Shift+K
  8. Verify nothing changes and diagnostics show "unsupported target"
  9. Enable auto mode in Switcher settings
  10. Type ghbdsn (with space) in the input field
  11. Verify auto-correction to привіт
  12. Type hello — verify no change

Notes

  • Chrome requires UI Automation accessibility to be active. It activates automatically when Switcher connects via UIA client.
  • If UIA ValuePattern is not available on first try, wait 1 second and retry (Chrome may need a moment).
  • SetValue replaces the entire field value. Cursor position may reset to end after correction. This is a known limitation of the UIA adapter.
  • Auto mode in browser fields is experimental. Manual safe mode (hotkeys) is more reliable.
